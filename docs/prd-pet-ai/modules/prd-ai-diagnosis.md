## 1. 模块概述

### 1.1 模块职责

AI 初诊是本产品的核心差异化功能。用户通过文字描述宠物症状，AI（基于 LLM + 症状决策树）给出健康状态评估（三级分级）和行动建议，帮助宠物主在就医决策上做出更理性的判断。所有输出内容定位为"健康建议"而非"医疗诊断"，附带法律免责声明。

### 1.2 模块范围

| 包含 | 不包含 |
|------|--------|
| 文字症状描述问答 | 图像识别诊断（MVP 排除，后续迭代） |
| AI 健康评估结果（三级分级）| 正式医疗诊断或处方开具 |
| 行动建议（就医/观察/保健）| 在线兽医实时咨询（P2，独立模块） |
| 历史诊断记录管理 | 外部医院预约跳转 |
| 诊断结果→保健品推荐触发 | 药品销售推荐（合规风险） |

### 1.3 相关模块

| 模块 | 标识 | 关系 | 说明 |
|------|------|------|------|
| 宠物档案 | `pet-profile` | 上游依赖 | 读取宠物品种/年龄/体重作为 AI 上下文 |
| 保健品推荐 | `health-recommendation` | 下游消费 | 初诊完成后触发推荐模块展示相关保健品 |
| 电商购物 | `ecommerce` | 下游消费 | 推荐的保健品可直接加购 |

### 2.2 功能详细描述

#### 2.2.1 症状文字问答

- **描述**：用户通过自然语言描述宠物症状，系统引导用户提供完整信息，AI 进行评估
- **输入**：
  - 症状描述（文字，10-500字，必填）
  - 症状持续时间（选择：刚出现/1天内/2-3天/超过3天）
  - 严重程度自评（选择：轻微/明显/严重）
  - 当前宠物档案（自动绑定，可临时切换）
- **处理逻辑**：
  1. 检查症状描述长度（< 10 字提示"请详细描述症状"）
  2. 构建 Prompt：将宠物信息（品种/年龄/体重/绝育状态）+ 症状描述 + 持续时间 + 严重程度组装为 LLM 请求
  3. 症状决策树预处理：对关键词（呕吐/血便/抽搐/昏迷等）进行快速分类，触发对应的高风险提示
  4. 调用 LLM API（OpenAI GPT-4o），设置 Timeout 10 秒
  5. 解析 LLM 输出，结构化为评估结果格式
  6. 将本次诊断记录存入历史（绑定宠物 ID、时间戳）
- **输出**：跳转至 AI 健康评估结果页
- **异常场景**：
  - LLM 超时：显示"AI 服务繁忙，请稍后重试"，保留用户输入
  - LLM 返回无法解析：降级显示通用建议"症状描述较为复杂，建议咨询专业兽医"
  - 检测到高危关键词（抽搐/昏迷/大量出血）：跳过 LLM，立即显示红色"建议立即就医"

#### 2.2.2 AI 健康评估分级

- **描述**：将 AI 评估结果分为三个等级，直观展示健康状态
- **处理逻辑**：
  - 🟢 **正常**：宠物状态正常，可在家观察，无需特别处理
  - 🟡 **关注**：存在轻度不适，建议密切观察，如症状持续或加重则就医
  - 🔴 **就医**：存在明显症状，建议尽快预约就医
- **输出**：在结果页顶部以大号颜色标识展示评估等级
- **异常场景**：AI 无法判断时，显示"⚪ 无法评估"+ 建议咨询专业兽医

#### 2.2.3 行动建议生成

- **描述**：基于评估等级提供具体可操作的行动建议
- **处理逻辑**：
  - 🟢 正常：3条居家护理建议 + 日常保健品推荐入口
  - 🟡 关注：症状监测清单（哪些情况需立即就医）+ 居家缓解措施 + 针对性保健品推荐
  - 🔴 就医：就医时需告知兽医的关键信息清单 + 就医时注意事项
- **输出**：行动建议列表（3-5条）+ 推荐保健品入口（跳转保健品推荐模块）
- **异常场景**：无宠物档案时，建议内容使用通用版本，不包含品种特定信息

#### 2.2.4 历史诊断记录

- **描述**：用户可查看过去的所有 AI 初诊记录，追踪宠物健康变化
- **输入**：点击"历史记录"入口
- **处理逻辑**：
  1. 按时间倒序展示，每条记录显示：日期、宠物名、症状摘要（前50字）、评估等级
  2. 点击展开查看完整诊断详情
  3. 记录保留 12 个月，超期软删除
  4. 支持按宠物筛选
- **输出**：历史诊断列表
- **异常场景**：无历史记录时显示空状态插图 + "立即开始第一次问诊"按钮

## 3. 用户故事与验收标准

### 3.1 用户故事

| 编号 | 用户故事 | 验收标准 | 优先级 |
|------|----------|----------|--------|
| US-ai-diagnosis-001 | 作为宠物主，我想描述宠物症状并获得 AI 健康评估，以便决定是否需要就医 | 1. 症状提交后 ≤10 秒返回评估结果<br>2. 评估结果包含健康等级（绿/黄/红）<br>3. 包含 3 条以上行动建议 | P0 |
| US-ai-diagnosis-002 | 作为宠物主，我想在 AI 评估完成后直接看到相关保健品推荐，以便一次操作解决诊断和购买 | 1. 评估结果页底部展示推荐保健品（至少 3 款）<br>2. 推荐理由与症状/评估结果关联<br>3. 点击可直接跳转商品详情 | P0 |
| US-ai-diagnosis-003 | 作为宠物主，我想查看历史诊断记录，以便追踪宠物健康趋势 | 1. 历史记录按时间倒序展示<br>2. 每条记录可展开查看完整诊断<br>3. 支持按宠物筛选 | P1 |

### 4.1 关键交互流程

```
首页"开始问诊"按钮 → 选择/确认当前宠物 → 填写症状描述表单
→ 提交（等待中动画）→ AI评估结果页（等级 + 行动建议）
```

## 5. 依赖、风险与技术参考

### 5.1 外部依赖

| 依赖项 | 提供方 | 状态 | 影响范围 |
|--------|--------|------|----------|
| OpenAI GPT-4o API | OpenAI | 已确认（公开可用）| AI 评估核心能力 |
| 兽医顾问审核 | 合作兽医机构（待洽谈）| 待确认 | AI 输出内容合规审核 |

### 5.2 风险项

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| AI 健康建议合规风险 | 高 | 高 | 所有输出添加免责声明；定位为"辅助工具"；邀请执业兽医审核 Prompt 模板 |
| LLM 幻觉导致错误建议 | 中 | 高 | 症状决策树做高危词兜底；设置 AI 回答边界（不开具药方）；用户可举报不当内容 |
| OpenAI API 服务不可用 | 低 | 高 | 准备备用 LLM（如国内大模型）；服务降级时展示通用建议卡片 |
| API 成本超预期 | 中 | 中 | 对同类症状建立缓存；设置每用户每日问诊次数上限（免费5次/日） |
