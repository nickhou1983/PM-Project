## 1. 模块概述

### 1.1 模块职责

电商购物模块是产品的主要收入来源，提供完整的宠物用品购买闭环，包括商品搜索与浏览、商品详情、购物车、订单结算、支付和订单管理。通过与 AI 初诊和保健品推荐模块的无缝衔接，实现"诊断即购买"的差异化体验。

### 1.2 模块范围

| 包含 | 不包含 |
|------|--------|
| 商品搜索与分类浏览 | 直播购物 |
| 商品详情页（图片/规格/评价）| C2C 二手交易 |
| 购物车管理 | 团购/拼单 |
| 订单创建与支付（微信/支付宝）| 自建物流系统 |
| 订单管理（查询/取消/售后）| 供应商后台管理（独立系统） |
| 商品评价与评分 | 广告投放 |
| 收藏夹（P2）| 分期付款 |

### 1.3 相关模块

| 模块 | 标识 | 关系 | 说明 |
|------|------|------|------|
| 保健品推荐 | `health-recommendation` | 上游依赖 | 推荐的商品直接跳转至本模块的商品详情页 |
| 用户认证 | `user-auth` | 上游依赖 | 登录用户才能下单；收货地址绑定用户账号 |
| 宠物档案 | `pet-profile` | 双向协作 | 部分商品可与宠物档案关联（如购买记录加入健康档案） |

### 2.2 功能详细描述

#### 2.2.1 商品搜索与浏览

- **描述**：用户通过关键词搜索或分类导航找到心仪商品
- **输入**：
  - 搜索关键词（支持商品名/品牌/成分关键词）
  - 分类筛选（保健品/主食/零食/用品/清洁）
  - 排序方式（综合排序/销量/价格升降序）
  - 价格区间筛选
- **处理逻辑**：
  1. 关键词搜索：全文搜索商品名称、品牌、标签字段
  2. 搜索结果分页（每页 20 条）
  3. 展示商品缩略图、名称、价格、销量、评分
- **输出**：商品列表（分页）
- **异常场景**：搜索无结果时显示相关推荐商品；搜索词包含违禁词时清空并提示

#### 2.2.2 商品详情页

- **描述**：展示商品完整信息，支持规格选择和加购操作
- **输入**：商品 ID（URL 参数）、选择规格（如重量/包装数量）
- **处理逻辑**：
  1. 展示商品主图轮播（支持缩放）、文字描述、规格选择器
  2. 库存实时查询，库存 ≤5 件显示"仅剩 X 件"
  3. 展示用户评价（默认显示最新 5 条，可加载更多）
  4. 展示"其他用户还购买了"关联商品
- **输出**：商品详情页，底部固定加购和立即购买按钮
- **异常场景**：规格未选择时点击加购显示提示；商品下架显示"商品已下架"并推荐替代商品

#### 2.2.3 购物车与结算

- **描述**：用户管理待购商品，完成下单支付流程
- **输入**：商品 ID + 规格 + 数量（加购）；收货地址；支付方式（微信支付/支付宝）
- **处理逻辑**：
  1. 购物车支持增减数量（最大 99）、删除、全选/单选
  2. 实时计算合计金额
  3. 结算时校验库存（锁库存）
  4. 生成订单后跳转支付宝/微信支付 SDK
  5. 支付成功回调更新订单状态
  6. 支付超时（30 分钟）自动释放库存，订单取消
- **输出**：订单创建成功，跳转订单详情页（含物流单号，发货后更新）
- **异常场景**：
  - 库存不足：弹窗提示并自动减至库存量上限
  - 支付失败：保留订单 30 分钟，显示重试按钮
  - 网络异常：防重复提交（客户端锁 + 服务端幂等）

#### 2.2.4 订单管理

- **描述**：用户查看订单状态、物流信息，申请退款/售后
- **输入**：订单列表筛选（全部/待付款/待发货/待收货/已完成/售后）
- **处理逻辑**：
  1. 订单按创建时间倒序展示
  2. 物流信息调用第三方快递 API 实时查询（发货后）
  3. 已完成订单支持申请退款（7 天内）
  4. 退款审核通过后原路退款（3-5 个工作日）
- **输出**：订单列表 + 订单详情（商品/金额/物流/状态）
- **异常场景**：物流信息查询失败时显示"物流信息更新中，请稍后查看"

#### 2.2.5 商品评价

- **描述**：用户对购买并收货的商品进行评分和图文评价
- **输入**：星级评分（1-5星）、评价文字（可选，10-500字）、评价图片（可选，最多3张）
- **处理逻辑**：
  1. 仅已收货订单的商品可评价
  2. 每个商品每次购买只能评价一次
  3. 评价内容进行违禁词过滤
  4. 评价成功后更新商品平均评分
- **输出**：评价提交成功，显示在商品详情页
- **异常场景**：评价包含违禁词时提示"评价包含不合规内容，请修改"

## 4. 交互设计与原型

### 4.1 关键交互流程

```
商品搜索/分类浏览 → 商品详情页 → 选择规格 → 加入购物车
→ 购物车 → 确认收货地址 → 选择支付方式 → 调起支付 SDK → 支付成功 → 订单详情
```

```
AI推荐保健品卡片 → 点击"加购"→ 购物车（商品已添加）→ 去结算 → 支付
```

| 购物车结算页 | [wireframes/ecommerce-cart.html](../wireframes/ecommerce-cart.html) | 购物车商品列表、合计金额、去结算流程 |

## 5. 依赖、风险与技术参考

### 5.1 外部依赖

| 依赖项 | 提供方 | 状态 | 影响范围 |
|--------|--------|------|----------|
| 微信支付 SDK | 腾讯微信开放平台 | 待申请商户号 | 支付核心功能 |
| 支付宝 SDK | 蚂蚁集团 | 待申请商户号 | 支付核心功能 |
| 快递查询 API | 快递100/快递鸟 | 待确认 | 物流轨迹展示 |
| 商品供应商后台 | 3-5 家保健品供应商 | 待洽谈 | 商品数据、库存、发货 |

### 5.2 风险项

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 支付 SDK 审核周期长 | 高 | 高 | 提前申请微信/支付宝商户资质；开发阶段使用沙箱环境 |
| 供应商发货慢影响体验 | 中 | 高 | 在商品页明示发货时效；服务协议中约定发货时间 SLA |
| 库存超卖 | 中 | 高 | 下单时数据库悲观锁锁定库存；支付回调再次确认库存 |
| 退款纠纷 | 中 | 中 | 明确退换货政策；提供 7 天无理由退款 |

### 5.3 技术参考

> 此部分为占位区域，由 `architect` Skill 生成模块级架构设计后填充或链接。

- **模块级架构文档**：[architecture-pet-ai-ecommerce.md](../architecture-pet-ai-ecommerce.md)（若已生成）
- **涉及数据模型**：`products`（productId, name, category, tags, price, stock, images）、`orders`（orderId, userId, items, totalAmount, status, paymentMethod, createdAt）、`reviews`（reviewId, userId, productId, rating, content, images）
- **涉及 API 端点**：`GET /api/v1/products`（搜索）、`GET /api/v1/products/:id`（详情）、`POST /api/v1/cart/items`（加购）、`POST /api/v1/orders`（创建订单）、`GET /api/v1/orders`（订单列表）、`POST /api/v1/reviews`（提交评价）
- **涉及前端组件**：ProductCard、ProductDetail、CartList、CheckoutForm、OrderList、ReviewForm
