## 1. 模块概述

### 1.1 模块职责

场景发现模块负责帮助宠物主快速找到真正适合携宠出行的地点，解决“信息分散、规则不透明、与当前宠物不匹配”的问题。模块以地图为主视图，整合附近场所、结构化入宠规则、用户打卡评价与导航能力，并与宠物档案联动，形成更可信的带宠决策入口。

### 1.2 模块范围

| 包含 | 不包含 |
|------|--------|
| 基于地图的宠物友好场所检索与点位展示 | 自建地图引擎或路线规划算法 |
| 按宠物类型/体型/场景/设施进行筛选 | 即时聊天社交功能 |
| 场所详情、营业信息、入宠规则、设施标签 | 团购/到店券交易（后续商业化） |
| 跳转第三方地图导航 | 商家结算系统 |
| 用户打卡评价、图片上传、标签反馈 | 复杂 UGC 社区流（单独社区模块） |
| 商家提交场所信息的基础入口（P1） | 全量商家后台管理系统 |

### 1.3 相关模块

| 模块 | 标识 | 关系 | 说明 |
|------|------|------|------|
| 用户认证 | `user-auth` | 上游依赖 | 打卡、收藏、商家提交等行为需要登录态 |
| 宠物档案 | `pet-profile` | 双向协作 | 读取当前宠物信息，进行场所规则匹配和筛选预设 |
| AI 初诊 | `ai-diagnosis` | 下游协作 | AI 健康结果可影响推荐外出提示（如不建议剧烈活动） |
| 保健品推荐 | `health-recommendation` | 下游协作 | 场所详情可为出行用品推荐提供入口 |
| 电商购物 | `ecommerce` | 下游消费 | 从场景发现页导流宠物用品购买 |

## 2. 功能需求

### 2.1 功能概览

| 功能点 | 描述 | 优先级 | R | I | C | E | RICE 分 | 状态 |
|--------|------|--------|---|---|---|---|---------|------|
| 附近宠物友好地图 | 地图展示周边宠物友好场所点位、距离和聚合数量 | P0 | 10 | 3 | 80% | 2.0 | 12.0 | 待开发 |
| 场所详情与入宠规则 | 结构化展示场所信息、规则、设施、评价与匹配提示 | P0 | 9 | 3 | 90% | 1.5 | 16.2 | 待开发 |
| 条件筛选与列表联动 | 支持按宠物类型、体型、场景和设施筛选并与列表联动 | P0 | 9 | 2 | 80% | 1.5 | 9.6 | 待开发 |
| 地图导航跳转 | 跳转系统地图进行路线规划与导航 | P1 | 8 | 2 | 90% | 1.5 | 9.6 | 待开发 |
| 打卡评价 | 发布图片、评分和标签评价，沉淀 UGC 反馈 | P1 | 7 | 2 | 70% | 1.5 | 6.5 | 待开发 |

> **RICE 列说明**：R=Reach（影响面 1-10）、I=Impact（影响度 0.25-3）、C=Confidence（信心度 50%-100%）、E=Effort（工作量/人月）。RICE 分 = (R × I × C) / E。

### 2.2 功能详细描述

#### 2.2.1 附近宠物友好地图

- **描述**：用户进入模块后默认看到基于当前位置的宠物友好地图，按场所类型展示周边点位、距离和数量。
- **输入**：定位授权结果、当前城市、地图缩放层级、用户主动搜索城市/商圈。
- **处理逻辑**：
  1. 首次进入时说明定位用途并请求权限；拒绝后切换为手动选城模式。
  2. 根据地图中心点与缩放级别请求附近场所数据，并对点位进行聚合。
  3. 顶部展示当前宠物快捷切换，读取其类型/体型信息作为筛选预设。
  4. 支持按“咖啡馆 / 公园 / 商场 / 酒店 / 医院周边休息区”等场景切换。
- **输出**：地图点位、底部推荐卡片、附近场所数量、距离信息。
- **异常场景**：定位失败时展示手动选城入口；附近无结果时展示空状态和热门商圈推荐。

#### 2.2.2 场所详情与入宠规则

- **描述**：用户点击点位或列表项后进入详情页，查看场所基础信息、入宠规则、设施标签、评分与近期打卡。
- **输入**：场所 ID、当前宠物档案、用户评价数据。
- **处理逻辑**：
  1. 聚合场所基础信息（名称、营业时间、地址、联系电话、封面图占位）。
  2. 以结构化标签展示规则（是否需牵引、是否限制体型、是否需疫苗证明、室内外可进入区域）。
  3. 若当前宠物档案与规则冲突，展示“谨慎前往”提醒。
  4. 评价区优先展示“是否适合大型犬/猫咪”“店员态度”“拥挤程度”等标签化反馈。
- **输出**：场所详情页、规则标签、匹配提醒、导航按钮、打卡入口。
- **异常场景**：规则缺失时标记“待确认”；场所已停业时显示下线提示并提供纠错反馈入口。

#### 2.2.3 条件筛选与列表联动

- **描述**：用户可通过筛选面板限制结果范围，并在地图与列表之间联动切换。
- **输入**：宠物类型、宠物体型、场景类型、距离范围、设施标签、开放时间。
- **处理逻辑**：
  1. 筛选面板采用抽屉/底部弹层呈现，支持多选。
  2. 应用筛选后同步更新地图点位、场所列表和结果数量。
  3. 记忆用户最近一次筛选条件，便于下次快速恢复。
  4. 当定位不可用时，优先展示按商圈/区域排序的列表结果。
- **输出**：筛选标签、列表结果、地图点位联动。
- **异常场景**：条件过多导致无结果时提示清空筛选；数据加载失败时支持重试。

#### 2.2.4 地图导航跳转

- **描述**：用户在场所详情页可直接跳转系统地图应用，进行步行/驾车/打车导航。
- **输入**：场所经纬度、用户当前位置、首选导航方式。
- **处理逻辑**：
  1. 调起系统或第三方地图进行路线规划。
  2. 若目标场所停车或电梯信息已知，可在跳转前展示简短提示。
  3. 记录详情页到导航页的转化埋点。
- **输出**：成功跳转导航或提供复制地址/经纬度的备选方式。
- **异常场景**：系统无可用地图应用时，提供复制地址与电话咨询按钮。

#### 2.2.5 打卡评价

- **描述**：用户到店后可发布图片、评分和标签评价，帮助其他宠物主判断是否值得前往。
- **输入**：1-9 张图片、文字评价、评分、宠物适配标签、规则纠错信息。
- **处理逻辑**：
  1. 已登录用户方可发布打卡评价；未登录则引导登录。
  2. 支持标签化反馈（适合猫咪/大型犬、是否需推车、是否有饮水）。
  3. 图片上传完成后进入审核状态，文本通过基础内容安全校验。
  4. 打卡发布后更新场所详情页的评价摘要和计数。
- **输出**：打卡成功页、评价卡片、个人打卡记录。
- **异常场景**：图片上传失败时支持重传；违规内容进入审核失败并告知原因。

## 5. 依赖、风险与技术参考

### 5.1 外部依赖

| 依赖项 | 提供方 | 状态 | 影响范围 |
|--------|--------|------|----------|
| 高德地图 SDK / API | 高德 | 待确认 | 地图展示、定位、导航跳转 |
| 宠物友好场所数据源 | 本地商家 / 第三方 POI | 待确认 | 冷启动数据质量与覆盖率 |
| 图片存储与审核能力 | 阿里云 OSS / 内容安全服务 | 待确认 | 打卡图片上传与审核 |
| 电话/地址基础信息 | 商家提供 / 地图 POI | 待确认 | 场所详情页基础信息准确性 |

### 5.2 风险项

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 场所规则更新不及时导致用户误判 | 高 | 高 | 为规则字段增加更新时间、用户纠错入口和“待确认”标识 |
| 初期 POI 数量不足导致地图价值弱 | 高 | 高 | 聚焦核心城市和热点商圈；同时启动商家 BD 与用户共建 |
| UGC 打卡质量参差不齐 | 中 | 中 | 引入标签化模板、基础审核与举报机制，优先展示高质量内容 |
| 地图 API 调用成本超预期 | 中 | 中 | 做点位聚合、缓存热点区域结果，并设置调用配额监控 |
