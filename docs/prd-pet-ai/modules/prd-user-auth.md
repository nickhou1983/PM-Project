## 1. 模块概述

### 1.1 模块职责

用户认证模块负责用户注册、登录、身份验证及账号管理，是整个 App 的入口基础模块。通过简洁的手机号+验证码方式降低注册门槛，支持第三方登录提升体验，确保用户数据安全。

### 1.2 模块范围

| 包含 | 不包含 |
|------|--------|
| 手机号注册与登录（验证码） | 邮箱注册/登录 |
| 第三方登录（微信/Apple） | 密码登录（安全风险高，简化流程） |
| Token 管理与刷新 | 企业账号/多租户 |
| 账号注销与隐私数据删除 | 商家/供应商账号体系（独立后台处理） |
| 个人信息编辑（昵称/头像） | 社交功能（关注/粉丝） |

### 1.3 相关模块

| 模块 | 标识 | 关系 | 说明 |
|------|------|------|------|
| 宠物档案 | `pet-profile` | 下游消费 | 用户登录后才能创建和管理宠物档案 |
| AI 初诊 | `ai-diagnosis` | 下游消费 | 用户身份用于绑定诊断记录 |
| 电商购物 | `ecommerce` | 下游消费 | 用户身份用于订单、支付、地址管理 |

## 2. 功能需求

### 2.1 功能概览

| 功能点 | 描述 | 优先级 | R | I | C | E | RICE 分 | 状态 |
|--------|------|--------|---|---|---|---|---------|------|
| 手机号注册/登录 | 手机号+短信验证码一键注册与登录 | P0 | 10 | 2 | 100% | 0.5 | 40.0 | 待开发 |
| Token 管理 | JWT Token 签发、刷新与过期处理 | P0 | 10 | 2 | 100% | 0.5 | 40.0 | 待开发 |
| 个人信息编辑 | 修改昵称、头像、性别等基本信息 | P1 | 8 | 1 | 100% | 0.5 | 16.0 | 待开发 |
| 账号注销 | 用户主动注销账号并删除个人数据 | P1 | 5 | 1 | 100% | 0.5 | 10.0 | 待开发 |
| 第三方登录 | 微信/Apple 一键登录 | P2 | 7 | 0.5 | 80% | 0.5 | 5.6 | 待开发 |

> **RICE 列说明**：R=Reach（影响面 1-10）、I=Impact（影响度 0.25-3）、C=Confidence（信心度 50%-100%）、E=Effort（工作量/人月）。RICE 分 = (R × I × C) / E。

### 2.2 功能详细描述

#### 2.2.1 手机号注册/登录

- **描述**：用户输入手机号，系统发送 6 位数字验证码短信，验证成功后完成注册（新用户）或登录（老用户），无需区分注册/登录流程
- **输入**：手机号（11位大陆号码）、短信验证码
- **处理逻辑**：
  1. 校验手机号格式（正则）
  2. 调用短信服务发送验证码，有效期 5 分钟，同一手机号 60 秒内不可重发
  3. 验证码匹配：匹配成功则签发 JWT Access Token（有效期 7 天）+ Refresh Token（有效期 30 天）
  4. 新用户首次登录自动创建账号，默认昵称"宠物主 + 随机4位数字"
- **输出**：Access Token、用户基本信息（userId、昵称、头像）
- **异常场景**：
  - 验证码错误：提示"验证码错误，请重新输入"，最多允许 5 次错误，超过后锁定 30 分钟
  - 手机号格式错误：实时校验提示
  - 短信发送失败：提示"短信发送失败，请稍后重试"并记录日志

#### 2.2.2 Token 管理

- **描述**：维护用户登录状态，处理 Token 过期与无效的情况
- **输入**：Access Token（HTTP Header）
- **处理逻辑**：
  1. 每次 API 请求验证 Access Token 有效性
  2. Access Token 过期时，客户端自动使用 Refresh Token 获取新 Access Token
  3. Refresh Token 过期则强制重新登录
- **输出**：新 Access Token（刷新时）/ 401 错误码（Token 无效）
- **异常场景**：Refresh Token 过期返回 401，客户端跳转登录页

#### 2.2.3 个人信息编辑

- **描述**：用户可修改昵称、头像、性别
- **输入**：昵称（1-20字）、头像图片（≤5MB，支持 JPG/PNG）、性别（可选）
- **处理逻辑**：头像上传到 OSS，存储 URL；昵称去除首尾空格；校验昵称不含违禁词
- **输出**：更新后的用户信息
- **异常场景**：图片格式/大小不符提示具体要求；违禁词提示"昵称包含不合规内容"

#### 2.2.4 账号注销

- **描述**：用户主动注销账号，删除所有个人数据（符合《个人信息保护法》）
- **输入**：用户确认（二次确认弹窗）+ 验证码校验
- **处理逻辑**：
  1. 展示注销影响说明（订单数据/宠物档案将被删除）
  2. 手机验证码二次确认
  3. 软删除账号数据，30 天冷静期后永久删除
  4. 使所有 Token 立即失效
- **输出**：注销成功提示，跳转登录页
- **异常场景**：有未完成的订单时，提示"存在未完成订单，暂时无法注销"

## 3. 用户故事与验收标准

### 3.1 用户故事

| 编号 | 用户故事 | 验收标准 | 优先级 |
|------|----------|----------|--------|
| US-user-auth-001 | 作为新用户，我想用手机号验证码注册，以便快速开始使用 App | 1. 输入手机号后 3 秒内收到验证码短信<br>2. 验证码正确后自动创建账号并进入首页<br>3. 新用户注册成功后引导创建宠物档案 | P0 |
| US-user-auth-002 | 作为老用户，我想用同一手机号快速登录，以便无需记住密码 | 1. 登录与注册共用同一入口（无需区分）<br>2. 验证成功后 1 秒内进入首页<br>3. 7 天内无需重新登录（Token 自动刷新） | P0 |
| US-user-auth-003 | 作为用户，我想修改我的昵称和头像，以便个性化我的账号 | 1. 昵称修改实时生效<br>2. 头像上传完成后立即在个人中心显示 | P1 |
| US-user-auth-004 | 作为用户，我想注销账号并删除我的数据，以便保护我的隐私 | 1. 注销前展示数据删除范围说明<br>2. 需通过手机验证码二次确认<br>3. 注销后 30 天内可恢复账号 | P1 |

| 短信验证码被刷 | 高 | 中 | 增加图形验证码前置、IP 频率限制、手机号黑名单机制 |
