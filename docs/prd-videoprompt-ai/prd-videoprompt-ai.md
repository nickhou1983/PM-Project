## 1. 背景与目标

### 1.1 项目背景

2026 年，AI 视频生成市场进入爆发期。OpenAI Sora 2 和字节跳动 Seedance 2.0 成为两大主流文生视频模型，大量内容创作者、自媒体人和营销从业者开始使用 AI 生成短视频。

然而，写好 AI 视频提示词（Prompt）的门槛极高：用户需理解运镜术语（如推轨镜头、摇臂、航拍）、光影描述（如伦勃朗光、逆光剪影）、构图原则等专业知识。同时，Sora 和 Seedance 的提示词格式和最佳实践存在显著差异，用户在两个平台间切换的学习成本很高。

当前市场上缺乏一个专门的智能提示词编辑器，能够同时适配 Sora 和 Seedance，并通过结构化交互 + AI 智能扩写，帮助普通用户快速写出专业级视频提示词。

### 1.2 业务目标

1. **降低 AI 视频提示词创作门槛**：让不懂专业运镜/光影术语的用户也能写出高质量提示词
2. **打造 Sora + Seedance 双模型适配的第一工具**：抢占市场空白，建立先发优势
3. **构建提示词工具型 SaaS 产品**：通过 Freemium 模式实现商业化
4. **快速验证市场需求**：2 周内上线 MVP，获取用户反馈数据

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| MVP 上线后首月 DAU | ≥ 500 | 前端埋点统计 |
| 提示词生成次数/用户/天 | ≥ 3 | 后端日志统计 |
| 7 日留存率 | ≥ 30% | 用户行为分析 |
| 免费→付费转化率 | ≥ 3%（上线 3 个月后） | 支付系统统计 |
| AI 扩写功能使用率 | ≥ 60% | 功能埋点 |

### 4.2 功能详细描述

#### 4.2.1 结构化参数面板

- **所属模块**：提示词编辑器
- **描述**：提供可视化的参数选择面板，用户通过下拉菜单、标签选择等方式设置视频提示词的各维度参数，系统实时组合为完整提示词文本
- **参数维度**：
  - **主体描述**：文本输入（如"一个穿红色连衣裙的女孩"）
  - **动作**：文本输入（如"在雨中奔跑"）
  - **环境/场景**：下拉 + 文本（城市街道/森林/海边/室内/自定义）
  - **运镜方式**：下拉选择（推轨/摇臂/航拍/手持/静态/环绕/跟随 等，附中文说明）
  - **光影**：下拉选择（自然光/黄金时段/伦勃朗光/逆光/霓虹灯/烛光 等）
  - **情绪/氛围**：标签选择（欢快/忧伤/紧张/宁静/史诗/神秘 等）
  - **视频风格**：下拉选择（电影感/纪录片/MV/广告/动画/赛博朋克 等）
  - **时长**：滑块（4s / 8s / 10s / 16s）
  - **画面比例**：选择（16:9 / 9:16 / 1:1）
- **输入**：用户交互选择/输入各参数
- **处理逻辑**：根据选中参数，按「主体 + 动作 + 环境 + 运镜 + 光影 + 情绪 + 风格」结构拼接提示词
- **输出**：实时更新的提示词文本预览
- **异常场景**：必填项（主体描述）为空时，提示用户填写

#### 4.2.2 AI 一键优化

- **所属模块**：AI 智能扩写
- **描述**：用户输入简单的场景描述或已拼接的粗糙提示词，AI 将其扩写为包含丰富视觉细节、运镜指令和光影描述的专业提示词
- **输入**：用户输入的文本（≥ 5 个字符）
- **处理逻辑**：
  1. 调用 LLM API（OpenAI/Claude），传入用户文本 + 目标模型（Sora/Seedance）+ 系统 Prompt
  2. 系统 Prompt 包含视频提示词最佳实践规则（基于 OpenAI Sora Prompting Guide 和 Seedance 官方手册）
  3. 返回扩写后的完整提示词
- **输出**：AI 生成的优化提示词，用户可直接采用或继续编辑
- **异常场景**：
  - 输入过短（< 5 字符）：提示"请输入更详细的描述"
  - API 调用失败：显示错误提示并支持重试
  - API 响应超时（> 10s）：显示加载动画并提示"生成中，请稍候"

#### 4.2.3 Sora 格式输出

- **所属模块**：模型适配
- **描述**：将编辑器中的提示词按 Sora 2 的最佳实践格式化输出
- **格式规则**（基于 OpenAI 官方 Prompting Guide）：
  - 使用英文输出（自动翻译）
  - 按「画面主体 + 具体动作 + 环境细节 + 运镜指令 + 光影描述 + 画面风格」结构
  - 动作描述具体化（避免 "moves quickly"，改用 "jogs three steps and stops"）
  - 环境描述使用可见的物理细节（避免 "beautiful street"，改用 "wet asphalt, neon signs reflecting in puddles"）
  - 技术参数放在最后（如 "anamorphic 2.0x lens, shallow DOF"）
- **输出**：格式化的 Sora 提示词文本

#### 4.2.4 Seedance 格式输出

- **所属模块**：模型适配
- **描述**：将编辑器中的提示词按 Seedance 2.0 的格式和最佳实践输出
- **格式规则**（基于即梦官方手册）：
  - 支持中文输出（Seedance 对中文支持良好）
  - 支持 @ 引用语法标注（如 @视频1、@图片1）
  - 运镜和转场描述符合 Seedance 的理解模式
  - 可附加参考视频/图片的引用说明
- **输出**：格式化的 Seedance 提示词文本


## 7. 技术方案概要

### 7.1 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Next.js 14 + React 18 + TypeScript | SSR/SSG 支持 SEO，React 生态成熟，TypeScript 保障代码质量 |
| UI 框架 | Tailwind CSS + shadcn/ui | 高效开发 + 可定制组件库 |
| 后端 | Next.js API Routes (Serverless) | 前后端一体，MVP 快速交付，无需独立后端服务 |
| AI 服务 | OpenAI API (GPT-4o) | 用于 AI 扩写和提示词优化 |
| 数据库 | Supabase (PostgreSQL) | 免费额度充足，支持用户认证 + 数据存储 |
| 认证 | Supabase Auth (OAuth 2.0) | 支持 Google/GitHub 登录，与数据库无缝集成 |
| 部署 | Vercel | 与 Next.js 深度集成，全球 CDN，免费额度适合 MVP |
| 本地存储 | localStorage | 未登录用户的历史记录和偏好存储 |

### 7.2 系统架构概述

```
用户浏览器 (Next.js SSR)
    ├── 编辑器页面（客户端渲染，实时交互）
    ├── 模板库页面（SSG 静态生成，SEO 友好）
    └── 用户中心页面（需认证）

Next.js API Routes (Serverless Functions)
    ├── /api/ai/enhance     → 调用 OpenAI API 扩写提示词
    ├── /api/ai/translate    → 调用 OpenAI API 翻译提示词
    ├── /api/prompts/*       → CRUD 用户提示词（Supabase）
    └── /api/auth/*          → 用户认证（Supabase Auth）

Supabase (PostgreSQL + Auth)
    ├── users 表
    ├── prompts 表（用户保存的提示词）
    └── templates 表（预设模板数据）
```

### 7.3 关键接口定义

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| AI 扩写 | POST | /api/ai/enhance | 输入简单描述 + 目标模型，返回优化后的提示词 |
| AI 翻译 | POST | /api/ai/translate | 输入提示词 + 目标语言，返回翻译结果 |
| 保存提示词 | POST | /api/prompts | 保存提示词到用户个人库 |
| 获取提示词列表 | GET | /api/prompts | 获取当前用户的提示词列表（分页） |
| 删除提示词 | DELETE | /api/prompts/:id | 删除指定提示词 |
| 获取模板列表 | GET | /api/templates | 获取模板列表（支持分类筛选） |

## 10. 术语表

| 术语 | 定义 |
|------|------|
| Prompt（提示词） | 用户输入给 AI 模型的文字描述，用于指导 AI 生成视频内容 |
| Sora | OpenAI 推出的文生视频 AI 模型，支持从文字描述生成高质量视频 |
| Seedance | 字节跳动推出的 AI 视频生成模型（即梦平台），支持多模态参考输入 |
| 运镜（Camera Movement） | 摄影术语，描述摄像机的运动方式，如推轨、摇臂、航拍、手持等 |
| 扩写（Enhance/Expand） | AI 将简短描述扩展为包含丰富细节的完整提示词的过程 |
| 结构化提示词 | 按固定维度（主体+动作+环境+运镜+光影+情绪）组织的提示词格式 |
| @ 引用语法 | Seedance 特有的提示词格式，用 @视频1、@图片1 引用参考素材 |
| Freemium | 免费增值商业模式，基础功能免费，高级功能付费 |
| MVP | Minimum Viable Product，最小可行产品，用于快速验证市场需求 |
| RICE | 功能优先级评估框架：Reach（影响面）× Impact（影响度）× Confidence（信心度）÷ Effort（工作量） |
